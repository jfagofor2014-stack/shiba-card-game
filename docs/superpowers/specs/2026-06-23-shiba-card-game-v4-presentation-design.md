# 柴犬カードゲーム v4 演出・横画面 設計書

作成日: 2026-06-23

実プレイのフィードバックを受けた演出強化。対象は既存アプリ（`~/shiba-card-game`、CPU・パス&プレイ・オンライン稼働中、17種52枚）。

## 目的（6項目）

1. 先攻・後攻を半自動で決める「しっぽクジ」演出
2. カード選択後のカードバトル演出
3. スマホ横画面に最適化（横画面レイアウト）
4. スマホを横画面に（半）強制
5. 勝敗決着の派手な演出（紙吹雪等）
6. 結果画面で「もう一回？」再戦（3モード）

方針：演出は表示層（新規 `effects.js` ＋ CSS）に閉じ、ゲーム判定ロジック（game-rules.js）は原則変更しない（先攻のランダム化のみ小変更）。Android中心だが iOS でも破綻しない共通方式。

## 1. 横画面の（半）強制

- アプリ起動後の最初のユーザー操作（モード選択ボタン等のタップ）で、ベストエフォートで全画面＋横ロックを試行：
  - `document.documentElement.requestFullscreen()`（失敗は握りつぶす）
  - `screen.orientation.lock('landscape')`（Android Chromeで有効。未対応環境は例外を握りつぶす）
- **回転案内オーバーレイ**：`window.matchMedia('(orientation: portrait)')` が真の間、全面オーバーレイ「📱↻ 横にしてください」を表示し操作をブロック。横向きになると自動で消える（iOS含め確実に効く保険）。`orientationchange`/`resize`/matchMedia change で表示を更新
- アプリ全体（メニュー・遊び方・対戦・結果）を横画面前提でレイアウト

## 2. 横画面レイアウト最適化

- 対戦画面（`data-screen="game"`）を横長に再構成：
  - 上部：細い帯に「手番表示＋スコア＋直前の出来事」（横幅を使って1行）
  - 中段：相手の場 / 自分の場（薄い横帯）。クンクン公開時の相手手札もここ
  - 下部：手札を**横一列**に大きく並べる（`overflow-x:auto`、カードは横画面の高さに合わせて拡大）
  - 縦スクロール無しで横1画面に収める
- メニュー・ガイド・結果画面も横画面で中央に収まるよう調整
- 既存の縦向きCSSは横向き前提に置き換え（`@media (orientation: landscape)` を主、portrait時は回転案内が出るので最小限）

## 3. 先攻決定「しっぽクジ」演出

- 各モードのゲーム開始時に**先攻決定画面**を挟む（`data-screen="lottery"` 等）。現行の「host固定先攻」を置き換え
- 演出：2本の柴犬のしっぽ🐕が左右で揺れ、「クジを引く」ボタン→しっぽが回り**ランダムに片方が巻く（クルン）**→巻いた側が先攻。結果を「◯◯の先攻！」と表示してゲーム開始
- 「半自動」＝ボタンで発動・抽選結果は自動。先攻は**ランダム**（50/50）
- プレイヤー表記：CPU=あなた/CPU、パス&プレイ=プレイヤー1/2、オンライン=あなた/あいて
- ゲームロジック側：`createInitialState(rng, firstPlayer)` のように先攻を指定できるよう小変更（デフォルト 'host' で既存テスト維持）。抽選結果の役（host/guest）を渡して初期 `turn` を設定。オンラインは部屋作成側が抽選し state に反映

## 4. カードバトル演出

- カード確定時、約0.7秒の演出：対象カードが中央にせり出して拡大→一閃（フラッシュ/インパクト）→効果ポップ（「スキ+2！」が弾け、スコアがカウントアップ）→カードが場/捨て札へ
- カウンター時：攻撃カードと拒否柴/虚無顔がぶつかる演出＋「無効化！」表示
- 種別の軽い差分（任意・初版は共通演出＋少数差分）：ズーミーズ成功/失敗、無限換毛期の毛が舞う
- テンポ重視：短く、画面タップで**スキップ可**（連打で飛ばせる）。3モード共通
- 演出は表示のみ。state 更新は既存ロジックのまま、演出後に盤面を再描画

## 5. 勝敗の派手な演出

- 勝利：**紙吹雪（confetti、CSSアニメ）** ＋大きく「勝ち！🎉」＋勝った柴犬がぴょんぴょん跳ねる。スキ20到達の瞬間に発火
- 敗北（パス&プレイ/オンラインの負け視点）：「おしい！またね🐕」と優しめ
- 外部ライブラリ不要（CSS/JS自前）。音は無し

## 6. 「もう一回？」再戦（3モード）

- 結果画面に「もう一回 🔁」と「トップに戻る」
- **CPU**：同じ難易度で即再戦（しっぽクジから）
- **パス&プレイ**：しっぽクジ→再戦
- **オンライン**：両者が「もう一回」を押したら再戦。部屋 state に `rematch: { host:false, guest:false }` を持たせ、押した側を true に。両方 true で新しい対局（`createInitialState`）を生成し、抽選結果（先攻）も含めて同期、両端末でゲーム再開。片方待ちは「相手の返事待ち…」表示。トップに戻る/離脱時は rematch をリセット

## アーキテクチャ・実装インパクト

- **新規 `src/effects.js`**：純UI演出関数（`showLottery(labels, onResult)`、`playCardBattle(cardId, onDone)`、`showWinCelebration(winnerLabel)`、`requestLandscape()`、回転案内の制御）。DOM操作のみ、ゲーム状態は持たない
- **index.html**：演出用のDOM（回転案内オーバーレイ、しっぽクジ画面、バトル演出レイヤ、紙吹雪コンテナ、結果画面の「もう一回」ボタン）
- **style.css**：横画面レイアウト（`@media (orientation: landscape)`）、回転案内、しっぽクジ・バトル・紙吹雪のアニメ（@keyframes）
- **src/game-rules.js**：`createInitialState` に先攻指定の引数を追加（デフォルト host、既存テスト不変）。他は不変
- **src/main.js**：フロー統合＝モード選択→（横ロック試行）→しっぽクジ→ゲーム（各プレイで `playCardBattle`）→20到達で `showWinCelebration`→結果画面→「もう一回」で各モードの再戦。オンラインは rematch 同期と抽選結果の伝播
- **src/online.js**：rematch フラグの読み書き、再戦時の新 state 書き込み

## テスト方針

- **単体（game-rules）**：`createInitialState` の先攻指定（host/guest で初期 turn が変わる、デフォルトは host で既存通り）。合計枚数等は不変。既存 59 テスト維持
- **演出（effects.js）**：純UIで自動テストは最小限（関数が例外なく要素を生成/除去する程度）。中心はブラウザ実機確認
- **ブラウザ実機**：横画面でのレイアウト（縦スクロール無し）、回転案内（縦で出て横で消える）、しっぽクジで先攻がランダムに決まる、カードバトル演出が出てスキップできる、20到達で紙吹雪、結果画面の「もう一回」が3モードで機能（CPU/パス即再戦、オンラインは両者同意で再戦）、既存のCPU/パス/オンラインのゲーム進行が壊れていない

## スコープ外（YAGNI）

- 効果音（無し）
- カード個別の凝った専用アニメ（共通演出＋少数差分にとどめる）
- オンラインの既知の制約（切断時デッドロック等）やログ視点問題は引き続き据え置き
- 新カードのイラスト（別途）
