# 『我が道を行く！至高の柴犬ライフ』オンライン対戦アプリ 設計書

作成日: 2026-06-22

## 1. 概要

柴犬テーマの2人用カードゲームをスマホで遊べるWebアプリにする。
当面は夫婦（2人）での対戦が主目的だが、ルームコード方式で将来の拡張も可能。
加えて1人用のCPU対戦モードを備える。

- 勝利条件: 満足度（ポイント）を先に **20ポイント** 集めたら勝ち
- プレイ人数: 2人（オンライン対戦）／ 1人（CPU対戦）
- プレイ時間: 10〜15分
- カード: 全40枚

## 2. 全体構成

```
[スマホA] ──┐
            ├── Firebase Realtime Database（部屋の状態を同期）
[スマホB] ──┘

フロント   : Vanilla JS + HTML/CSS（過去アプリと同じ構成）
ホスティング: GitHub Pages
PWA対応    : ホーム画面に追加して全画面プレイ
バックエンド: Firebase Realtime Database（無料枠、サーバーコード不要）
```

- オンライン対戦: 片方が「部屋を作る」→ 4桁コード発行 → もう片方がコード入力で参加 → 開始
- CPU対戦: Firebase不要・1端末完結（オフライン可）。同一ロジックをAIが操作
- 全ゲーム状態（山札・手札・場・得点・手番）はFirebase上の1部屋に保存し両端末が購読
- カード見た目: 絵文字 ＋ カテゴリ色分け ＋ CSSカード枠（後でイラストを差し込める構造）

## 3. カード仕様（全40枚）

| カード名 | カテゴリ | 枚数 | 効果 |
| --- | --- | --- | --- |
| ヒコーキ耳 | 甘え | 6 | 満足度を2ポイント獲得 |
| へそ天 | 甘え | 4 | 満足度を3ポイント獲得し、山札から1枚引く |
| 拒否柴（不動柴） | 拒否 | 6 | 相手が得点獲得しようとした際、割り込んで無効化（0点）にできる |
| 虚無顔 | 拒否 | 4 | 相手の妨害カード（手札破壊・1回休み等）を割り込んで無効化できる |
| 柴距離 | マイペース | 6 | 満足度を1ポイント獲得し、相手の手札をランダムに1枚捨てさせる |
| 隙間にすっぽり | マイペース | 4 | 場に出すと次の自分のターン開始時まで相手の効果を受けない（ターン開始時に捨てる） |
| 柴ドリル | 野生 | 4 | 手札を好きな枚数捨て、捨てた枚数＋1枚を山札から引く |
| ズーミーズ（柴走り） | 野生 | 4 | 山札トップをめくり「甘え」なら5点獲得、それ以外はターン強制終了 |
| 無限換毛期 | 野生 | 2 | 相手を1回休み（ターンスキップ）にする |

カードは40枚それぞれに固有ID（例 `hikoki_1`〜`hikoki_6`）を振り、種類は定義テーブルで管理。

カテゴリ色分け（暫定）: 甘え=ピンク / 拒否=ブルー / マイペース=グリーン / 野生=オレンジ。

## 4. データ構造（Firebase上の「部屋」）

```
rooms/{4桁コード}/
  status:    "waiting" | "playing" | "finished"
  createdAt: タイムスタンプ
  players:
    host:  { name, connected }
    guest: { name, connected }
  state:
    deck:      [カードID, ...]        # 山札（順番あり）
    discard:   [カードID, ...]        # 捨て札
    hands:
      host:  [カードID, ...]
      guest: [カードID, ...]
    field:
      host:  [カードID, ...]          # 例: 隙間にすっぽり
      guest: [カードID, ...]
    scores:    { host: 0, guest: 0 }
    turn:      "host" | "guest"
    phase:     "main" | "awaiting_counter" | "zoomies_reveal"
    pending:                          # 割り込み待ちの行動内容
      actor, cardId, effect, targetScore など
    winner:    null | "host" | "guest"
  log: [ "ホストがヒコーキ耳で2点獲得", ... ]
```

補足:
- 手札は相手のぶんもDB上にあるため技術的には覗き見可能だが、夫婦戦なので信頼前提でOK（暗号化はしない＝YAGNI）
- 山札が尽きたら捨て札をシャッフルして再生成

## 5. ターンの流れと割り込み処理（状態機械）

通常ターン（phase: `main`）:
1. 手番プレイヤーが手札から1枚タップ → 確認 → 場に出す
2. そのカードが得点 or 妨害を含む場合 → phase を `awaiting_counter` にし相手の応答待ちへ
3. 相手が該当カウンターを持っていれば「○○が出されました！無効化しますか？[はい/いいえ]」ポップアップ（時間無制限）
   - はい → 元の効果を無効化、カウンターカードを捨て札へ、ログ記録
   - いいえ / カウンター不所持 → 効果を確定実行
4. 効果解決後、手札が5枚になるまで補充 → 手番交代（phase を `main` へ）
5. どちらかが20点到達 → status `finished`、勝者表示

特殊フェーズ:
- ズーミーズ（phase: `zoomies_reveal`）: 山札トップをめくる演出 → 「甘え」なら+5点、それ以外はターン強制終了
- 隙間にすっぽり: 場に出すと次の自分のターン開始時まで相手の効果を受けない。割り込み判定・効果適用前に「対象が隙間にすっぽりを場に出しているか」を確認して無効化。ターン開始時に場から捨てる
- 虚無顔: 妨害カード（柴距離の手札破壊・無限換毛期の1回休み）へのカウンターとして `awaiting_counter` で処理

重要な原則:
- 状態変更は必ず「手番側の端末が書き込む」方式にして同時書き込みの競合を防ぐ
- 割り込み時だけ相手端末が `pending` に応答を書き込む

## 6. CPU対戦モード

- トップ画面に「CPUと対戦」ボタン（オンライン対戦とは別の入口）
- Firebase不要・1端末完結（オフライン可）。同一ゲームロジックをAIが操作
- 割り込みもCPUが自動判断
- 難易度2段階を選択画面で提供:
  - **やさしい**: ほぼランダムに合法手を選ぶ。カウンターも消極的
  - **ふつう**: 定石ベース。得点カードは積極的に、相手が20点に近ければ妨害、損しない範囲でカウンター

## 7. コード構成

```
game-rules.js   … 純粋なゲームロジック（カード定義・効果解決・勝敗判定）。通信も画面も触らない
ai.js           … CPUの思考（やさしい/ふつう）。game-rulesの合法手から選ぶ
online.js       … Firebase接続・部屋管理・状態同期
ui.js           … 画面描画・タップ操作・ポップアップ
main.js         … 画面遷移（トップ→モード選択→対戦→結果）
index.html / style.css / manifest.json（PWA）
```

## 8. テスト方針

- `game-rules.js` は副作用がないので、各カード効果・カウンター・20点勝利・山札再生成を単体テストで検証（テスト駆動で実装）
- AIとロジックを繋ぎ「CPU vs CPU を最後まで自動で回す」テストでルールが破綻なく1ゲーム完走することを確認
- オンライン同期とUIは実機（2台のスマホ）で動作確認

## 9. 画面遷移

```
トップ画面
 ├─「CPUと対戦」→ 難易度選択（やさしい/ふつう）→ 対戦画面 → 結果画面
 └─「オンライン対戦」→ 部屋を作る / 参加する（コード入力）→ 待機 → 対戦画面 → 結果画面
```
