# 柴犬カードゲーム v5 ターンタイマー＆コンボ演出 設計書

作成日: 2026-06-23

対象は既存アプリ（`~/shiba-card-game`、CPU・パス&プレイ・オンライン稼働中、17種52枚、演出v4実装済み）。

## 目的（2機能）

1. **10秒ターンタイマー**：人間の手番に10秒のカウントダウン。0秒で手札からランダムに1枚自動プレイ。
2. **コンボ演出**：「おかわり」連鎖で1ターンに複数枚出したとき、相性の良い組み合わせに「○○コンボ！」を演出表示（効果は変えない、見た目のみ）。

方針：演出・UIは表示層（`src/effects.js` ＋ CSS ＋ `main.js` のフロー）に閉じる。ゲーム判定ロジック（`game-rules.js`）は変更しない。

## 1. 10秒ターンタイマー

- 人間が「カードを出せる状態」（手番側で `phase==='main'`）になったら、画面に10秒カウントダウンを表示（残り秒数の数字＋バー）。残り3秒で赤く点滅
- **0秒で手札からランダムに合法手を1枚自動プレイ**（通常プレイと同じ経路＝バトル演出付き）。`legalPlays(state, role)` からランダム選択し、そのモードのプレイ処理を呼ぶ
- **おかわりで追加行動になったら、その都度10秒にリセット**（次の1枚にまた10秒）
- 適用範囲：CPU戦の人間（host）／パス&プレイの手番側（holder）／オンラインの自分（myRole）。**CPUの手番はタイマー無し**
- カウンター確認（はい/いいえ）は手番ではないのでタイマー対象外
- タイマー停止：カードを出した瞬間、勝敗確定、自分の手番でなくなった瞬間、画面遷移時
- ドリルカードの自動プレイ時は `drillDiscard: []`（既存の人間プレイと同じ簡略仕様）

### タイマーの起動/停止ポイント

- 起動：各モードで「人間が手札をタップできる状態を描画した直後」。具体的には CPU `refresh`（turn===HUMAN && phase==='main' && !winner）、パス `renderPass`（turn===holder && phase==='main' && !winner）、オンライン `renderOnline`（turn===myRole && phase==='main' && !winner）
- 停止：プレイ確定（onPlayCard系の冒頭で stop）、勝敗確定、手番でない描画時、画面遷移

## 2. コンボ演出（おかわり連鎖中）

1ターン中に出したカードの種類を記録し（ターン交代＝endTurn でリセット）、相性の良い組み合わせが成立したら「○○コンボ！」を約0.8秒演出表示（バトル演出の後に重ねる）。**効果は変わらない（見た目のみ）**。

| コンボ名 | 条件（同一ターン中に出したカード kind の集合/列） |
| --- | --- |
| 🔄 引き直しコンボ | `dassou` を出した後に `kuidame` を出す |
| 👃 のぞき見コンボ | `kunkun` を出した後に `nusumi`/`yakimochi`/`itazura` のいずれかを出す |
| 😈 いじわるコンボ | 同一ターンに `nusumi` と `yakimochi` の両方を出す |
| 🩷 甘えんぼコンボ | 同一ターンに amae カテゴリ（`hikoki`/`hesoten`/`kuidame`）を2枚以上出す |
| 💨 ノンストップコンボ | 同一ターンに3枚以上出す（おかわり連鎖） |

- 同じコンボは1ターンに1回だけ発火（重複表示しない）。複数コンボが同時成立しうるが、1回のプレイで新たに成立したコンボを順に表示（最大でも数個）
- おかわりが無ければ1ターン1枚なのでコンボは出ない（おかわり時のご褒美演出）

## アーキテクチャ・実装インパクト

- **src/effects.js**:
  - `startTurnTimer(seconds, onTick, onTimeout)` — カウントダウンを開始。毎秒 `onTick(remaining)`、0で `onTimeout()`。既存タイマーがあれば破棄
  - `stopTurnTimer()` — 停止＆タイマーUIを隠す
  - `renderTimer(remaining)` — タイマーUIの表示（残り秒・バー・残り3秒で点滅クラス）。onTick から使う、または startTurnTimer 内で完結
  - `showCombo(name, onDone)` — 「{name}コンボ！」を約0.8秒表示（バトル演出レイヤとは別、または流用）
- **index.html**: タイマー表示要素（`#turn-timer`）、コンボ演出要素（`#combo-layer` か battle-layer 流用）
- **style.css**: タイマーバー/点滅、コンボ表示のアニメ
- **src/main.js**:
  - タイマー：各モードの人間手番描画で `startTurnTimer(10, onTick, onTimeout)`、プレイ時/非手番で `stopTurnTimer()`。onTimeout は `legalPlays` からランダムな cardId を選び、そのモードの onPlayCard 系を呼ぶ
  - コンボ：module 変数 `turnPlays = []`（出した kind の列）。各モードでカードのプレイが確定（resolve）したら kind を push し、コンボ判定して新規成立分を `showCombo` 表示。`endTurn` を呼ぶ箇所で `turnPlays = []` にリセット（コンボ既発火セットもリセット）
- **src/game-rules.js**: 変更なし

## テスト方針

- **コンボ判定（純ロジック）**：コンボ判定関数を `effects.js` か小さな純関数として切り出し、`turnPlays` 列に対して成立するコンボ名の配列を返すことを単体テスト（例：['dassou','kuidame']→引き直しコンボ、['kunkun','nusumi']→のぞき見、3枚→ノンストップ）。`test/` に追加
- **タイマー**：純粋なカウントダウンロジック（残り秒の減少・0でコールバック）を単体テスト可能な形にするか、ブラウザ実機で確認（数字が減る、3秒で点滅、0で自動プレイ、おかわりでリセット）
- **ブラウザ実機**：CPU戦でタイマーが10→0で自動プレイ、カードを出すと止まる、おかわりでリセット。おかわりを使って複数枚出しコンボ演出が出る。既存のCPU/パス/オンライン進行・v4演出が壊れていない
- 既存60テストは維持
