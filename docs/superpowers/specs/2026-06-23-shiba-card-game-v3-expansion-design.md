# 柴犬カードゲーム v3 カード拡張 設計書

作成日: 2026-06-23

実プレイのフィードバック「カードの種類が少なく単調」を受けた拡張。対象は既存アプリ（`~/shiba-card-game`、CPU・パス&プレイ・オンライン稼働中、イラスト版v3公開済み）。

## 目的

新しい特殊カードを8種追加し、9種40枚 → **17種52枚**に拡張。効果の方向性は「相手のスキを奪う/減らす」「コンボ・連鎖」「情報・かく乱」。勝利ラインは20のまま。バランスはCPU対CPUシミュレーションで実測して確定。

イラストは後で用意する方針。新カードはまず**絵文字＋カテゴリ色**で実装し、既存の `.card-art` 差し込み口に後からイラストを足せる構造を維持する。

## 効果の詳細仕様

- **盗み食い (nusumi)**: `score` 攻撃。確定実行時、自分 `addScore(+1)`、相手は `scores[opp] = max(0, scores[opp]-1)`。相手が隙間にすっぽり中なら相手−1は無効（自分+1は入る）。拒否柴で無効化されると自分+1も相手−1も発生しない（得点獲得の無効化）。
- **ヤキモチ (yakimochi)**: `sabotage`。確定実行時、相手 `scores[opp] = max(0, scores[opp]-2)`。相手が隙間にすっぽり中なら無効。虚無顔で無効化可。
- **イタズラ (itazura)**: `sabotage`。相手の手札からランダムに最大2枚を捨て札へ。相手が隙間にすっぽり中なら無効。虚無顔で無効化可。
- **脱走 (dassou)**: 割り込み無し。自分の手札を全て捨て札へ送り、`drawCards(self, 5)` で5枚引く（山札が尽きたら既存ロジックで捨て札再シャッフル）。
- **クンクン (kunkun)**: 割り込み無し。`state.reveal[who] = true`（次の自分のターン開始時=endTurnでこのプレイヤーの手番が来たとき解除）を立て、相手手札を可視化。さらに1枚引く。
- **物々交換 (kokan)**: 割り込み無し。自分の手札からランダム1枚と相手の手札からランダム1枚を入れ替える。相手が隙間にすっぽり中なら無効。どちらかの手札が0枚なら何もしない。
- **食いだめ (kuidame)**: `score` 攻撃。確定実行時、`addScore(self, min(4, 手札枚数))`。手札枚数は「このカードを場に出した後（捨て札へ送った後）」の残り手札で数える。拒否柴で無効化可。
- **おかわり (okawari)**: 割り込み無し。確定実行時 `state.extraActions = (state.extraActions||0) + 1`。ターン終了処理は追加行動が残っている間は手番を交代せず、同じプレイヤーがもう1枚プレイする。

## ターン進行の変更（おかわり＝追加行動）

- state に `extraActions`（数値、初期0）を追加。
- カードの効果解決後（割り込み解決まで含めて完了したあと）、手番側に `extraActions > 0` が残っていれば、`extraActions` を1減らして**手番を終えず同じプレイヤーがもう1枚プレイ**する。
- `extraActions === 0` になったら通常どおり `endTurn`（手札補充・手番交代）。
- 3モードでの扱い：
  - CPU戦：人間の追加行動は手札クリックを継続受付、CPUの追加行動は `cpuTurn` を再度回す。
  - パス&プレイ：追加行動中は同じプレイヤーが続行するため**受け渡し画面を出さない**（holder不変）。
  - オンライン：actor が手番を継続（追加行動中は endTurn せず、自分の番のまま次の手をプレイして push）。
- おかわりを2枚連続で使えば追加行動が累積し3枚出せる。

## 情報公開（クンクン）

- state に `reveal: { host: false, guest: false }` を追加。
- クンクンを出すと `reveal[who] = true`。`endTurn` で who の手番が再び来たとき（=次の自分の番開始）に `reveal[who] = false` へ戻す。実装上は endTurn 完了後に手番が who に戻った時点で解除、または「自分のターン開始処理」で解除する。
- UI: `renderBoard(state, viewer, ...)` で、`viewer` の `reveal` が true の場合、相手の手札を表向き（カード画像）で表示する。通常は相手手札は枚数のみ（既存は相手手札を描画していないので、公開時のみ相手手札エリアに表示する）。

## アーキテクチャ・実装インパクト

- **src/game-rules.js**:
  - `CARD_TYPES` に8種追加（name/category/count/emoji/color）。
  - `resolveEffect` に8効果を追加。負の得点・手札破壊・交換・引き直し・追加行動フラグ・公開フラグを実装。`addScore` は20以上で勝利のまま（負の加算では勝利判定に影響しないが、クランプ `max(0, …)` を相手スコアに適用）。
  - `needsCounter`: `score` に nusumi・kuidame、`sabotage` に yakimochi・itazura を追加。
  - `createInitialState` に `extraActions: 0` と `reveal: { host:false, guest:false }` を追加。
  - 追加行動の進行は、ターン終了の手前で extraActions を確認する責務を持つヘルパー（例 `consumeExtraAction(state)`）か、各モード制御側で判定。ロジックの一貫性のため game-rules 側に「追加行動が残るか」を判定する純粋関数を置く。
- **src/ai.js**: chooseMain/chooseCounter に新カードの価値・優先度を追加（盗み食い・食いだめ・ヤキモチ・イタズラは攻撃/妨害として、相手が高得点なら優先。おかわりは他に強い手があるときの起点。脱走は手札が弱いとき。クンクン・物々交換は低優先のユーティリティ）。AI は追加行動でも合法手を選べること。
- **src/ui.js**: 新カードの絵文字＋効果テキスト＋カテゴリ色。クンクン時に相手手札を表示する描画。`EFFECT_TEXT` に8種追加。
- **src/main.js**: おかわりの追加行動を3モードで処理（上記「ターン進行の変更」）。
- **画像/SW**: 新カードのイラストは後日。`sw.js` のキャッシュ資産は新イラスト追加時に更新（今回は絵文字なので画像追加なし、VERSION は実装反映時に bump）。

## テスト方針

- **単体テスト（test/game-rules.test.js）**: 各新効果（盗み食いの+1/−1とクランプ、ヤキモチの−2クランプ、イタズラの2枚破壊、脱走の引き直し、物々交換の交換、食いだめの手札枚数得点、おかわりの追加行動フラグ）、隙間にすっぽりによる無効化、拒否柴/虚無顔による割り込み無効化。合計枚数が52であること。
- **完走テスト（test/engine.test.js）**: 追加行動を含むCPU対CPVが多シードで無限ループ・例外なく完走し勝者が出ること。エンジンドライバが追加行動に対応すること。
- **バランス（scripts/balance-sim.mjs）**: 統計レポートで最終枚数を決定。
- **UI/モード**: ブラウザ実機（新カードの表示、クンクンで相手手札が見える、おかわりで連続プレイ＝パス&プレイで受け渡しが入らない、各効果が見た目通り動く）。
